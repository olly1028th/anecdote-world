# Anecdote World - 여행 트래커 기획서

> "Life is short and the world is wide."

## 1. 프로젝트 개요

**내가 다녀온 여행**과 **앞으로 갈 여행**을 한눈에 관리하는 개인 여행 웹페이지입니다.

| 구분 | 완료한 여행 | 계획 중인 여행 |
|------|------------|--------------|
| 핵심 정보 | 경비, 사진, 일정 | 예상 경비, 추천 장소 |
| 상태 | 기록 보관용 | 준비/계획용 |

---

## 2. 페이지 구성 (어떤 화면이 필요한가?)

웹페이지는 크게 **4개 화면**으로 나뉩니다.

### 2-1. 메인 페이지 (홈)
```
┌─────────────────────────────────────────┐
│  Anecdote World                         │
│  "Life is short and the world is wide." │
├─────────────────────────────────────────┤
│                                         │
│  [세계 지도 또는 여행 목록]               │
│   - 다녀온 곳: 핀 표시 (색상 A)          │
│   - 계획 중인 곳: 핀 표시 (색상 B)        │
│                                         │
│  ┌──────────┐  ┌──────────┐             │
│  │ 완료 여행  │  │ 계획 여행  │             │
│  │  5개      │  │  3개      │             │
│  └──────────┘  └──────────┘             │
└─────────────────────────────────────────┘
```
- 전체 여행 수 요약 (완료 몇 개, 계획 몇 개)
- 여행 카드 목록 (클릭하면 상세 페이지로 이동)

## 3. 핵심 기능 목록

### 기능을 이해하기 쉽게 "할 수 있는 것" 기준으로 정리했습니다

#### A. 여행 관리 (CRUD)
| 기능 | 설명 | 비유 |
|------|------|------|
| 여행 추가 | 새로운 여행 정보를 입력 | 다이어리에 새 페이지 만들기 |
| 여행 수정 | 기존 여행 정보를 고치기 | 다이어리 내용 수정하기 |
| 여행 삭제 | 여행 기록 지우기 | 다이어리 페이지 찢기 |
| 여행 조회 | 여행 목록/상세 보기 | 다이어리 넘겨보기 |

> **CRUD란?** Create(만들기), Read(읽기), Update(수정), Delete(삭제)의 약자입니다.
> 대부분의 웹 서비스는 이 4가지 동작이 기본입니다.

#### B. 완료 여행 전용 기능
| 기능 | 설명 |
|------|------|
| 사진 업로드/갤러리 | 여행 사진을 올리고 갤러리 형태로 보여주기 |
| 일정 타임라인 | Day 1, Day 2... 형태로 일정 보여주기 |
| 경비 기록 | 항공/숙박/식비/기타 항목별 금액 입력 |
| 경비 합계 | 입력한 금액을 자동으로 합산 |
| 한줄 후기 | 짧은 감상 메모 |

#### C. 계획 여행 전용 기능
| 기능 | 설명 |
|------|------|
| 예상 경비 | 항목별 예상 금액 입력 |
| 추천 장소 목록 | 가볼 곳 리스트 관리 (추가/삭제) |
| 준비 체크리스트 | 할 일 목록 (체크 가능) |
| 상태 변경 | 계획 → 완료로 전환 (여행 다녀오면) |

#### D. 공통 기능
| 기능 | 설명 |
|------|------|
| 여행 목록 보기 | 카드 형태로 전체 여행 나열 |
| 완료/계획 필터 | 탭이나 버튼으로 구분해서 보기 |
| 반응형 디자인 | 모바일/PC 모두에서 보기 좋게 |

## 4. 사용자 시나리오

> **사용자 시나리오란?** "사용자가 실제로 어떤 순서로 웹사이트를 쓸까?"를 글로 풀어 쓴 것입니다.
> 이걸 먼저 정리해두면 "어떤 기능이 진짜 필요한지" 명확해집니다.

### 시나리오 1: 다녀온 여행 기록하기

```
민수는 지난달 도쿄 여행을 다녀왔다.
추억을 정리하고 싶어서 Anecdote World에 접속한다.

1. 메인 페이지에서 [+ 새 여행 추가] 버튼을 누른다
2. 상태를 "완료"로 선택한다
3. 여행지 "도쿄", 기간 "2025.03.15 ~ 2025.03.20"을 입력한다
4. 사진 10장을 업로드한다
5. Day별 일정을 입력한다
   - Day 1: 시부야 → 하라주쿠 → 시부야 스카이
   - Day 2: 아사쿠사 → 스카이트리
   - ...
6. 경비를 항목별로 입력한다
   - 항공 350,000원 / 숙박 400,000원 / 식비 200,000원 / 기타 150,000원
7. 한줄 후기: "벚꽃 시즌 최고였다!"
8. [저장] 버튼을 누른다

→ 결과: 메인 페이지에 도쿄 여행 카드가 추가되고,
   클릭하면 사진/일정/경비가 예쁘게 보인다.
```

### 시나리오 2: 여행 계획 세우기

```
민수는 여름에 로마에 가고 싶다.
미리 계획을 정리하려고 Anecdote World에 접속한다.

1. [+ 새 여행 추가] 버튼을 누른다
2. 상태를 "계획 중"으로 선택한다
3. 여행지 "로마", 예상 기간 "2026.06" 을 입력한다
4. 예상 경비를 입력한다
   - 항공 800,000원 / 숙박 600,000원 / 식비 300,000원
5. 추천 장소를 추가한다
   - 콜로세움, 바티칸 미술관, 트레비 분수, 판테온
6. 준비 체크리스트를 만든다
   - 여권 유효기간 확인 / 항공편 예약 / 숙소 예약 / 여행자 보험
7. [저장] 버튼을 누른다

→ 결과: 메인 페이지에 "계획 중" 표시가 된 로마 여행 카드가 보인다.
```

### 시나리오 3: 계획 여행을 완료로 바꾸기

```
민수가 드디어 로마 여행을 다녀왔다!

1. 메인 페이지에서 "로마 여행" 카드를 클릭한다
2. [수정] 버튼을 누른다
3. 상태를 "계획 중" → "완료"로 변경한다
4. 실제 일정과 경비를 입력한다
5. 사진을 업로드한다
6. 한줄 후기를 작성한다
7. [저장] 버튼을 누른다

→ 결과: 로마 여행이 "완료" 목록으로 이동하고,
   사진/일정/경비가 모두 보인다.
```

### 시나리오 4: 여행 기록 둘러보기

```
민수는 심심한 주말, 지난 여행들을 구경하고 싶다.

1. Anecdote World에 접속한다
2. 메인 페이지에서 "완료" 탭을 눌러 다녀온 여행만 본다
3. 도쿄 여행 카드를 클릭해서 사진과 일정을 구경한다
4. "계획 중" 탭으로 바꿔서 앞으로의 여행을 확인한다
5. 전체 경비를 보며 "나 여행에 이만큼 썼구나" 확인한다

→ 결과: 지난 추억도 되새기고, 다음 여행도 기대된다!
```

## 5. 데이터 구조 (어떤 정보를 저장해야 하는가?)

> **데이터 구조란?** "각 여행마다 어떤 항목을 저장할지" 정리한 것입니다.
> 노트에 표를 만든다고 생각하면 됩니다.

### 여행 (Trip)
```
여행 1건 = {
  id:          고유번호 (자동 생성)
  title:       여행 제목         예) "도쿄 벚꽃 여행"
  destination: 여행지            예) "도쿄, 일본"
  status:      상태              "completed" 또는 "planned"
  startDate:   시작일            예) "2025-03-15"
  endDate:     종료일            예) "2025-03-20"
  coverImage:  대표 이미지       예) "tokyo-cover.jpg"
  memo:        한줄 후기         예) "벚꽃 시즌 최고였다!"
  expenses:    경비 목록         (아래 참고)
  itinerary:   일정 목록         (아래 참고) - 완료 여행용
  photos:      사진 목록         (아래 참고) - 완료 여행용
  places:      추천 장소 목록    (아래 참고) - 계획 여행용
  checklist:   준비 체크리스트   (아래 참고) - 계획 여행용
  createdAt:   생성 시각
  updatedAt:   수정 시각
}
```

### 경비 (Expense)
```
경비 1건 = {
  category: 분류     "flight" | "hotel" | "food" | "transport" | "activity" | "other"
  amount:   금액     예) 350000
  label:    설명     예) "인천-나리타 왕복"
}
```

### 일정 (Itinerary) - 완료 여행용
```
일정 1건 = {
  day:          몇째 날    예) 1
  title:        제목       예) "시부야 & 하라주쿠"
  description:  상세 내용  예) "시부야 스카이 전망대 → 하라주쿠 거리 구경 → 저녁 라멘"
}
```

### 추천 장소 (Place) - 계획 여행용
```
장소 1건 = {
  name:       이름       예) "콜로세움"
  priority:   중요도     "must" | "want" | "maybe"
  note:       메모       예) "오전에 가야 줄이 짧다"
}
```

### 체크리스트 (Checklist) - 계획 여행용
```
항목 1건 = {
  text:      내용       예) "여권 유효기간 확인"
  checked:   완료 여부  true 또는 false
}
```

## 6. 추천 기술 스택 (초보자 기준)

> **기술 스택이란?** 웹사이트를 만들 때 사용하는 도구 모음입니다.
> 요리에 비유하면 "어떤 칼, 냄비, 오븐을 쓸 건지" 정하는 것입니다.

### 프론트엔드 (사용자가 보는 화면)
| 도구 | 역할 | 왜 이걸 쓰나? |
|------|------|-------------|
| **React** | 화면 만들기 | 가장 많이 쓰이고 배울 자료가 많음 |
| **TypeScript** | 코드 안전망 | 오타/실수를 미리 잡아줌 (초보일수록 도움됨) |
| **Tailwind CSS** | 디자인 | class 이름만으로 디자인 가능, CSS 파일 따로 안 만들어도 됨 |
| **React Router** | 페이지 이동 | 여러 페이지 간 이동 처리 |

### 데이터 저장
| 도구 | 역할 | 왜 이걸 쓰나? |
|------|------|-------------|
| **JSON 파일** (1단계) | 데이터 저장 | 가장 단순, 서버 없이 시작 가능 |
| **LocalStorage** (1단계) | 브라우저 저장 | 서버 없이 브라우저에 데이터 저장 |

> 처음에는 LocalStorage로 시작하고, 나중에 서버/DB를 붙이는 것을 추천합니다.

### 빌드 도구
| 도구 | 역할 |
|------|------|
| **Vite** | 프로젝트 생성/실행 (빠르고 설정 간단) |

## 7. 개발 순서 (추천 로드맵)

> 한번에 다 만들면 막막하니까, **단계별로 나눠서** 만드는 걸 추천합니다.

### Phase 1: 기본 뼈대
- Vite + React + TypeScript 프로젝트 생성
- 메인 페이지 레이아웃 만들기
- 여행 카드 컴포넌트 만들기
- 하드코딩된 샘플 데이터로 화면 확인

### Phase 2: 데이터 연결
- 여행 추가 폼 만들기
- LocalStorage에 데이터 저장/불러오기
- 완료/계획 필터(탭) 기능
- 여행 수정/삭제 기능

### Phase 3: 상세 페이지
- 완료 여행 상세 (일정 타임라인, 경비 표)
- 계획 여행 상세 (추천 장소, 체크리스트)
- 경비 자동 합산

### Phase 4: 사진 기능
- 사진 업로드 기능
- 사진 갤러리 보기
- 대표 이미지 설정

### Phase 5: 꾸미기 & 개선
- 반응형 디자인 (모바일 대응)
- 애니메이션/전환 효과
- 다크 모드 (선택)

## 9. 용어 정리 (초보자용)

| 용어 | 쉬운 설명 |
|------|----------|
| **프론트엔드** | 사용자가 직접 보고 클릭하는 화면 부분 |
| **컴포넌트** | 화면을 이루는 작은 조각 (레고 블록) |
| **라우팅** | 페이지 간 이동 (URL이 바뀌는 것) |
| **상태(State)** | 화면에서 변하는 데이터 (입력값, 목록 등) |
| **LocalStorage** | 브라우저가 제공하는 작은 저장소 (새로고침해도 유지) |
| **CRUD** | 만들기/읽기/수정/삭제 - 웹 서비스의 기본 동작 |
| **반응형** | 화면 크기에 따라 레이아웃이 자동 조절되는 것 |
| **TypeScript** | JavaScript에 타입(숫자/문자 구분)을 추가한 언어 |
