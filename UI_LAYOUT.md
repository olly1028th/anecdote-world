# Anecdote World - UI 요소 리스트업 & 레이아웃 묘사

## 1. 전체 웹페이지에 필요한 UI 요소 리스트

### A. 공통 요소 (모든 페이지에 반복)

| # | 요소 | 역할 |
|---|------|------|
| 1 | **헤더(Header)** | 사이트 이름 + 슬로건, 어디서든 홈으로 돌아가는 링크 역할 |
| 2 | **내비게이션 바** | 현재 위치 표시 (홈 / 상세 페이지) |
| 3 | **반응형 컨테이너** | 콘텐츠 최대 너비 제한, 좌우 여백 자동 정렬 |

### B. 메인 페이지 (홈) 요소

| # | 요소 | 역할 |
|---|------|------|
| 4 | **히어로 섹션** | 페이지 제목("나의 여행 기록") + 부제 |
| 5 | **요약 통계 카드 3개** | "다녀온 여행 수 / 계획 중 수 / 총 경비"를 숫자로 요약 |
| 6 | **필터 탭 버튼** | "전체 / 완료 / 계획 중" 전환 버튼 (pill 형태) |
| 7 | **여행 카드(TripCard)** | 각 여행 1개를 카드 형태로 보여주는 핵심 UI |
| 8 | **카드 그리드** | 여행 카드들을 반응형 격자로 배치 (1열→2열→3열) |
| 9 | **빈 상태(Empty State)** | 여행이 없을 때 표시하는 안내 메시지 |

### C. 여행 카드(TripCard) 내부 요소

| # | 요소 | 역할 |
|---|------|------|
| 10 | **커버 이미지** | 여행지 대표 사진 (카드 상단) |
| 11 | **상태 뱃지(StatusBadge)** | "완료" 또는 "계획 중" 표시 (이미지 위에 오버레이) |
| 12 | **여행 제목** | 카드 제목 (예: "도쿄 벚꽃 여행") |
| 13 | **여행지 텍스트** | 도시/나라 (예: "도쿄, 일본") |
| 14 | **날짜 & 기간** | "2025.03.15 ~ 2025.03.20 (5박 6일)" |
| 15 | **경비 요약** | "총 경비: 1,200,000원" 한 줄 요약 |
| 16 | **한줄 후기** | 완료 여행만 — 짧은 감상 텍스트 |
| 17 | **준비도 프로그레스 바** | 계획 여행만 — 체크리스트 진행률 바 |

### D. 완료 여행 상세 페이지 요소

| # | 요소 | 역할 |
|---|------|------|

| 19 | **히어로 배너** | 커버 이미지 + 그라데이션 오버레이 + 제목/날짜 |
| 20 | **한줄 후기 인용문** | 파란 배경의 인용문 블록 |
| 21 | **일정 타임라인(Timeline)** | Day별 일정을 세로 타임라인으로 표시 |
| 22 | **타임라인 원형 번호** | 각 Day의 번호를 원형 뱃지로 표시 |
| 23 | **타임라인 연결선** | 번호 사이를 잇는 세로 선 |
| 24 | **경비 내역 테이블** | 항목별 금액 나열 + 하단 합계 |

### E. 계획 여행 상세 페이지 요소

## 2. UI 레이아웃 묘사

### 2-1. 전체 페이지 구조

```
┌──────────────────────────────────────────────────────────┐
│ [헤더 바]                                                │
│  좌측: 사이트명 "Anecdote World" (클릭→홈)                │
│  아래: 슬로건 "Life is short and the world is wide."      │
│  배경: 흰색, 하단에 얇은 회색 테두리로 본문과 구분          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  [본문 영역]                                              │
│  최대 너비 1152px, 좌우 자동 여백으로 중앙 정렬             │
│  배경: 아주 연한 회색 (#f8fafc)                            │
│                                                          │
│  (여기에 각 페이지별 콘텐츠가 들어감)                       │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**왜 이렇게?**
- 헤더는 항상 고정되어 있어서 어느 페이지에 있든 "홈으로 돌아가기"가 가능
- 본문 최대 너비를 제한해서 큰 모니터에서도 글이 너무 넓게 퍼지지 않음
- 연한 배경 위에 흰색 카드를 올리면 카드가 자연스럽게 떠 보이는 효과

│ [필터 탭]                                                │

│  비선택 탭: 흰 배경 + 회색 글씨                            │

│  │ └─────┘   │  │ └─────┘   │  │ └─────┘   │          │
│  │            │  │            │  │            │          │
│  │ 도쿄 벚꽃   │  │ 방콕 맛집   │  │ 로마 여행   │          │
│  │ 도쿄, 일본  │  │ 방콕, 태국  │  │ 로마, 이탈  │          │
│  │ 3/15~3/20  │  │ 1/10~1/14  │  │ 6/10~6/17  │          │
│  │ ──────── │  │ ──────── │  │ ──────── │          │
│  │총경비      │  │총경비      │  │예상경비     │          │
│  │1,200,000원 │  │  650,000원 │  │2,150,000원 │          │
│  │"벚꽃최고!" │  │"팟타이!"   │  │준비도 1/6█░│          │
│  └────────────┘  └────────────┘  └────────────┘          │
│                                                          │
│  반응형 동작:                                             │
│    모바일(~640px): 1열 (카드가 세로로 쌓임)                │
│    태블릿(640~1024px): 2열                               │
│    데스크톱(1024px~): 3열                                 │
│                                                          │
│  각 카드:                                                │
│    - 흰 배경, 둥근 모서리(16px), 미세한 그림자              │
│    - hover 시 그림자가 약간 강해짐 (클릭 가능 암시)         │
│    - 상단: 커버 이미지 (높이 192px, 꽉 채움)               │
│    - 이미지 좌상단: 상태 뱃지 (둥근 알약 형태)              │

│    - 하단 패딩 영역:                                      │
│      여행 제목 (볼드), 여행지 (회색 작은 글씨)              │
│      날짜 범위 + 기간 (좌우 양끝 정렬)                     │
│      구분선                                               │
│      경비 (라벨 좌측, 금액 우측 볼드)                      │
│      완료→한줄후기 / 계획→프로그레스바                     │
└──────────────────────────────────────────────────────────┘

**카드 하단이 상태에 따라 다르게 보이는 점이 핵심:**
- **완료 여행 카드**: 이탤릭체 한줄 후기가 따옴표와 함께 표시
- **계획 여행 카드**: "준비도 1/6" + 얇은 가로 바(주황색 진행률)

## 3. 디자인 원칙 요약

### 색상 체계
| 용도 | 색상 | 의미 |
|------|------|------|
| 완료 상태 | 초록 계열 (`emerald`) | "다 했다!" 완료의 느낌 |
| 계획 상태 | 주황 계열 (`amber`) | "진행 중" 주의/기대 |
| 주요 액션/강조 | 파란 계열 (`blue`) | 링크, 선택된 탭, 타임라인 |
| 필수 장소 | 빨간 계열 (`red`) | "반드시!" 우선순위 강조 |
| 배경 | 연회색 (`#f8fafc`) | 눈이 편한 기본 배경 |
| 카드 | 흰색 (`#ffffff`) | 배경과 대비로 떠 보이는 효과 |

### 타이포그래피
| 용도 | 크기 | 굵기 |
|------|------|------|
| 페이지 제목 | 30px | Bold |
| 카드 제목 | 18px | Semibold |
| 본문/설명 | 14px | Regular |
| 보조 텍스트 | 12px | Regular, 회색 |

### 간격 규칙
| 요소 | 간격 |
|------|------|
| 섹션 간 | 24px (세로) |
| 카드 간 | 24px (가로+세로) |
| 카드 내부 패딩 | 16~20px |
| 텍스트 행간 | 4~8px |

### 반응형 기준
| 화면 크기 | 카드 열 수 | 동작 |
|-----------|-----------|------|
| ~640px (모바일) | 1열 | 카드가 세로로 쌓임 |
| 640~1024px (태블릿) | 2열 | 2개씩 나란히 |
| 1024px~ (데스크톱) | 3열 | 3개씩 나란히 |
